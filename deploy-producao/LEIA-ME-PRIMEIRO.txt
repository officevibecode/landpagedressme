3. CONFIGURAR O BACKEND (OBRIGATORIO!)
================================================================================

Apos extrair os arquivos:

A) VIA SSH (Recomendado):
   
   1. Conecte ao servidor:
      ssh usuario@seuservidor.com
   
   2. Navegue ate a pasta API:
      cd /var/www/html/api
      (ou cd ~/public_html/api)
   
   3. Instale as dependencias:
      composer install --no-dev --optimize-autoloader
   
   4. Crie o arquivo .env:
      nano .env
   
   5. Cole este conteudo (SUBSTITUA as chaves LIVE):
      
      STRIPE_SECRET_KEY=sk_live_SUA_CHAVE_SECRETA_AQUI
      STRIPE_PUBLISHABLE_KEY=pk_live_SUA_CHAVE_PUBLICA_AQUI
      BASE_URL=https://dressme.tgoo.eu
   
   6. Salve (Ctrl+O) e saia (Ctrl+X)
   
   7. Configure permissoes:
      chmod 600 .env
      chown www-data:www-data .env

B) VIA cPanel/Plesk:
   
   1. Acesse Terminal no painel
   2. Execute: cd api && composer install --no-dev
   3. Use o File Manager para criar arquivo .env
   4. Cole o conteudo acima (com chaves LIVE)

================================================================================
4. OBTER CHAVES DO STRIPE
================================================================================

1. Acesse: https://dashboard.stripe.com/apikeys
2. Mude para modo LIVE (toggle no topo)
3. Copie:
   - Publishable key: pk_live_...
   - Secret key: sk_live_... (clique em "Reveal")

ATENCAO: 
- Use pk_live_ e sk_live_ (NAO pk_test_ ou sk_test_!)
- NUNCA exponha a SECRET_KEY publicamente

================================================================================
5. VERIFICAR SE FUNCIONA
================================================================================

1. Acesse seu site: https://dressme.tgoo.eu

2. Verifique se:
   ‚úì Site carrega sem erros
   ‚úì Console do navegador (F12) sem erros
   ‚úì CSS aplicado corretamente
   ‚úì Imagens carregam

3. Teste o Checkout:
   - Clique em "Quero o DressMe"
   - Deve redirecionar para Stripe
   - Use um cartao de teste ou real
   - Complete o pagamento
   - Deve voltar para a pagina de sucesso

4. Verifique logs se houver erro:
   tail -f /var/log/apache2/error.log
   (ou caminho do seu servidor)

================================================================================
7. TROUBLESHOOTING
================================================================================

Erro 500 na API:
  - Verifique logs do PHP
  - Confirme que composer install foi executado
  - Verifique se .env existe e tem as chaves corretas

Site nao carrega:
  - Verifique permissoes dos arquivos (644 para arquivos, 755 para pastas)
  - Confirme que extraiu na pasta correta
  - Verifique configuracao do servidor web

Checkout nao funciona:
  - Verifique se esta usando chaves LIVE
  - Confirme que o site esta em HTTPS
  - Veja console do navegador (F12)
  - Verifique Stripe Dashboard por erros
